01 — The Hypothesis

Do LLMs build internal world-models?

Large language models may be operating with something functionally equivalent to an internal model of reality: a compressed, weighted representation of mathematical structure, physical law, and logical relationship embedded in their weights.

Core claim: A model's score on FORGE is evidence—not proof—of how deeply it has internalized mathematical and physical structure, as distinct from how much relevant training data it has seen.

This remains a contested scientific claim. Mechanistic interpretability research suggests models develop hierarchically structured understanding during training. But opposing evidence shows that reasoning models suffer catastrophic accuracy collapse—rather than graceful degradation—when pushed slightly beyond their training distribution.

FORGE does not assert an answer. It produces clean, contamination-free evidence that bears directly on this question.

Falsifiability: The hypothesis would be disproven if models scoring highly on maximum difficulty tiers suffer catastrophic degradation when mathematical structure remains identical but semantic framing is inverted—proving reliance on surface-level pattern matching rather than internalized circuits.

03 — Scoring

The Complexity Cliff Index

A simple mean accuracy conflates easy interpolation with true world-modeling. FORGE uses a weighted geometric mean that exponentially rewards performance at higher complexity tiers, penalizing models that only succeed at memorizable depths.

C_c = Σ(αd · Sc,d) / Σ(αd)  α = 1.5

The Cliff Index reports the exact difficulty tier where accuracy drops more than 30% from the previous tier. This is the single most diagnostic number FORGE produces.

MetricWhat it measures
FORGE ScoreWeighted accuracy across all categories and tiers
Interpolation ScorePerformance on Easy/Medium (likely in training data)
Extrapolation ScorePerformance on Hard/Expert (statistically novel)
Cliff IndexFirst tier where accuracy drops >30%

Key insight: A model with high Interpolation but low Extrapolation is likely memorizing. A model maintaining high scores across both zones has evidence of generalized reasoning circuits.

04 — The Nerfed Model Problem

The first benchmark with a built-in audit trail.

AI labs face a quiet incentive problem. A model is released. It scores well on public benchmarks. The scores become marketing. Then, quietly, the model is updated. The published scores no longer reflect what users are actually running.

There is currently no clean way to detect this. Fixed benchmarks cannot distinguish between "the model got smarter" and "the model was tuned toward this specific benchmark distribution."

FORGE changes this.

1 Lab publishes: Model X, Seed 42, FORGE Score 0.847
2 Timestamp recorded: 2025-01-15
3 Six months later, anyone can run the same seed against the same endpoint
4 If the score has shifted, the shift is real and documented

FORGE does not accuse anyone of nerfing. It simply makes the question answerable for the first time.

Seed security

FORGE uses a 256-bit hash space (SHA-256). There are more possible problem sets than atoms in the observable universe. Pre-computation is physically impossible.

06 — Get started

Run FORGE in under a minute.

Note: The Researcher Tool requires the Python backend running locally. It is a frontend UI that connects to localhost:7860 — without the backend installed and running, the page will show errors when you try to run a benchmark.

Install

# Clone and install
$ git clone https://github.com/mohamedhossammohamed/FORGE.git
$ cd FORGE && pip install -r requirements.txt

Run via CLI

# Quick evaluation (~5 min, 250 questions)
$ python -m forge.cli run \
  --mode quick \
  --model qwen/qwen3-235b-a22b-thinking-2507 \
  --api-base https://openrouter.ai/api/v1 \
  --api-key sk-... \
  --seed 42

Run via web UI

# Launch the researcher tool
$ python -m forge.server
# Opens at http://localhost:7860/researcher.html
